🎓 Viva Preparation - AI Smart Interview System

Prepared by: Ashraf Ali | Project: AI Smart Interview System

📌 Section 1: Project Overview

Q1. What is AI Smart Interview System?
A. It is a full-stack web application that simulates real job interviews using Artificial Intelligence. It generates questions based on the user's resume, records voice answers, provides AI feedback, and tracks performance history.
Q2. What is the main objective of this project?
A. The main objective is to help job seekers practice interviews in a realistic AI-powered environment, get instant feedback, and improve their performance over time using data-driven insights.
Q3. What technologies did you use in this project?
A. Frontend: React.js, Vite, CSS. Backend: Node.js, Express.js. Database: MongoDB with Mongoose. AI: OpenAI GPT API. Voice: Web Speech API (STT & TTS). Authentication: JWT (JSON Web Tokens).
Q4. What are the main modules of this project?
A. 1) User Authentication (Login/Register), 2) Resume Upload & Parsing, 3) AI Interview Session, 4) Voice Interaction (Speech to Text), 5) AI Feedback & Scoring, 6) Interview History Dashboard, 7) Question Bank (Admin).

📌 Section 2: Frontend (React.js)

Q5. Why did you use React.js for the frontend?
A. React.js provides a component-based architecture which makes the UI modular and reusable. It has fast rendering using Virtual DOM, strong community support, and integrates well with REST APIs.
Q6. What is Vite and why did you use it?
A. Vite is a fast build tool for modern web applications. It provides instant server start, hot module replacement (HMR), and much faster build speeds compared to traditional tools like Create React App.
Q7. How does routing work in your React app?
A. We use React Router DOM for client-side routing. Routes are defined for pages like Login, Register, Dashboard, Interview, History, and Resume Upload. Protected routes redirect unauthenticated users to the login page.
Q8. How do you manage state in your application?
A. We use React's built-in useState and useEffect hooks for local component state. For shared state like user authentication, we use React Context API to pass data across components without prop drilling.
Q9. What is the History page in your project?
A. The History page shows all past interview sessions of a user. It displays each session's date, score, questions asked, user's answers, AI feedback, strengths, and weaknesses in an expandable card layout.

📌 Section 3: Backend (Node.js & Express)

Q10. Why did you use Node.js for the backend?
A. Node.js is non-blocking and event-driven, making it highly efficient for handling multiple simultaneous API requests. It uses JavaScript on both frontend and backend, which simplifies development.
Q11. What is Express.js and what role does it play?
A. Express.js is a minimal web framework for Node.js. It handles HTTP routing, middleware management, request/response handling, and connects the frontend with the database and OpenAI API.
Q12. What API routes did you create in the backend?
A. Main routes: /api/auth (login, register), /api/interview (start session, submit answer), /api/history (get past sessions), /api/resume (upload and parse resume), /api/dashboard (performance stats), /api/questions (admin question bank).
Q13. How did you secure your API routes?
A. We use JWT (JSON Web Token) authentication middleware. Every protected route checks for a valid token in the request header. Admin routes use an additional secret key validation for extra security.
Q14. What is middleware in Express.js?
A. Middleware are functions that execute between receiving a request and sending a response. Examples: cors() for cross-origin requests, express.json() for parsing JSON body, and our custom authMiddleware for JWT verification.

📌 Section 4: Database (MongoDB)

Q15. Why did you choose MongoDB over SQL databases?
A. MongoDB is a NoSQL database that stores data as flexible JSON-like documents. It is ideal for this project because interview data (questions, answers, feedback) has varying structures that don't fit well into fixed SQL table schemas.
Q16. What is Mongoose and why is it used?
A. Mongoose is an Object Data Modeling (ODM) library for MongoDB and Node.js. It provides schema validation, model creation, and a clean API to interact with MongoDB, making database operations more structured and reliable.
Q17. What are the main MongoDB collections (schemas) in your project?
A. 1) User: name, email, password (hashed), createdAt. 2) Interview: userId, questions[], answers[], feedback[], score, date. 3) Question: text, category, difficulty, expectedAnswer. 4) Resume: userId, parsedText, skills, experience.
Q18. How do you store passwords securely?
A. Passwords are never stored in plain text. We use bcrypt.js to hash passwords before saving to the database. During login, bcrypt.compare() verifies the entered password against the stored hash.

📌 Section 5: AI & OpenAI Integration

Q19. How does the AI generate interview questions?
A. We send a prompt to OpenAI GPT API containing the user's resume text and job role. The AI generates relevant, personalized interview questions based on the candidate's skills and experience.
Q20. How does the AI provide feedback on answers?
A. After the user answers a question (via voice or text), we send both the question and the user's answer to GPT. The model evaluates it and returns structured feedback including score, strengths, weaknesses, and suggestions.
Q21. What is a prompt in the context of GPT/OpenAI?
A. A prompt is the instruction or input text we send to the GPT model. We craft specific prompts that tell GPT to act as an interviewer, evaluate responses, and return feedback in a structured JSON format.
Q22. What is an API key and how did you manage it securely?
A. An API key is a secret credential to authenticate with the OpenAI service. We store it in a .env file (OPENAI_API_KEY) and use dotenv package to load it. The .env file is never committed to version control (added to .gitignore).

📌 Section 6: Voice Interaction

Q23. How did you implement voice input (Speech to Text)?
A. We used the Web Speech API's SpeechRecognition interface built into modern browsers. It converts the user's spoken words into text in real-time, which is then sent as the interview answer to the backend.
Q24. How does Text to Speech (TTS) work in your project?
A. We use the SpeechSynthesis API (browser built-in). The AI-generated questions and feedback are read aloud using window.speechSynthesis.speak(), making the interview feel like a real conversation.
Q25. What is the Web Speech API?
A. The Web Speech API is a browser-native API that provides two interfaces: SpeechRecognition (converting voice to text) and SpeechSynthesis (converting text to voice). It works without any external library for basic usage.

📌 Section 7: Authentication & Security

Q26. What is JWT (JSON Web Token)?
A. JWT is a compact, self-contained token for securely transmitting information between client and server. After login, the server signs a token with a secret key. The client sends this token in every subsequent request for authentication.
Q27. What is the structure of a JWT token?
A. A JWT has three parts separated by dots: Header.Payload.Signature. Header: algorithm type. Payload: user data (userId, email). Signature: ensures the token hasn't been tampered with.
Q28. What is CORS and why is it needed?
A. CORS (Cross-Origin Resource Sharing) is a browser security feature that blocks requests from different origins. Since our frontend (port 5173) and backend (port 5000) run on different ports, we enable CORS middleware in Express to allow these cross-origin requests.

📌 Section 8: Resume Parsing

Q29. How does resume upload work in your project?
A. Users upload their PDF resume through the frontend. The backend uses multer middleware to handle file uploads. The resume text is extracted and parsed to identify skills, experience, and education, which are then used to generate personalized interview questions.
Q30. What is multer?
A. Multer is a Node.js middleware for handling multipart/form-data, which is primarily used for file uploads. It processes the uploaded file and makes it accessible via req.file in Express route handlers.

📌 Section 9: Project Architecture & Deployment

Q31. Explain the folder structure of your project.
A. The project has two main folders: /frontend (React app with Vite) and /backend (Node.js/Express server). Backend has /routes, /models, /middleware, /controllers, and /config folders. Frontend has /src with /pages, /components, /context, and /services.
Q32. What is a REST API?
A. REST (Representational State Transfer) API is an architectural style for building web services. It uses standard HTTP methods: GET (fetch data), POST (create data), PUT (update data), DELETE (remove data). Our backend exposes REST endpoints that the frontend consumes.
Q33. How does the frontend communicate with the backend?
A. The frontend uses the Fetch API or Axios to make HTTP requests to the backend REST API. The backend processes the request, interacts with MongoDB and OpenAI, and sends back a JSON response which React renders in the UI.
Q34. What is .env file and why is it important?
A. The .env file stores environment variables like API keys, database connection strings, and JWT secrets. It keeps sensitive configuration out of the source code. The dotenv package loads these variables into process.env at runtime.
Q35. What future enhancements can be added to this project?
A. 1) Video interview with facial expression analysis. 2) Multi-language support. 3) Company-specific interview modes. 4) Detailed analytics dashboard with charts. 5) Peer-to-peer mock interview feature. 6) Mobile app using React Native. 7) Integration with LinkedIn for profile-based questions.
